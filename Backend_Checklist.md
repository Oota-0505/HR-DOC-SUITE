# HR DOC SUITE バックエンド・実装チェックリスト

## 1. 共通（フロントエンド制御）
- [ ] **ヘッダーナビの切り替え**: 
  - 未ログイン時： 「ログイン」を表示
  - ログイン時： 「マイページ」を表示

## 2. 新規登録機能
- [ ] **バリデーション**: ID（メールアドレス）に「@」が含まれていない場合は登録不可とする
- [ ] **DB連携**: 入力された内容が正確にデータベース（WPユーザーテーブル等）に保存されること
- [ ] **遷移制御**: 「登録」ボタン押下後、即座にマイページへ自動遷移すること

## 3. 認証機能（ログイン）
- [ ] **正規ログイン**: 新規登録したID・パスワードで正常にログインできること
- [ ] **エラーハンドリング**: 登録されていない情報、または誤った情報でのログインをブロックすること
- [ ] **管理者ルーティング**: 管理者アカウント（role_type=1）でログインした場合は、「ユーザー選択ページ」へ遷移すること

## 4. 管理機能
- [ ] **データ反映**: 管理者が「ユーザー選択」から特定のユーザーの情報を編集した場合、その内容が該当ユーザーのマイページに正しく反映されること

## 5. マイページ機能
- [ ] **金額計算**: プラン選択時、合計金額（税込）の計算ロジックに誤りがないこと
- [ ] **決済アクション**: 「決済」ボタン押下時に確認用のポップアップが表示されること
- [ ] **会員情報表示**: アカウント情報エリアに、登録時の内容が正しく表示されていること
- [ ] **DL機能**: PDFテンプレートが正常にダウンロードできること
- [ ] **連絡リンク**: リンク押下時にメーラーが直接起動すること
